## Why

当前侧边栏搜索只能按项目名过滤，无法检索历史聊天正文。随着会话数量增加，用户即使记得关键词，也很难从跨项目、跨 provider 的历史记录里找回某次讨论，更无法直接跳到命中的那条消息继续工作。

## What Changes

- 新增全局聊天全文搜索能力，覆盖 Claude 与 Codex 的历史会话正文，而不只搜索项目名或会话摘要。
- 新增统一的后端搜索结果接口，由服务端执行搜索并返回项目、provider、session、命中消息、命中文字片段与定位信息。
- 在聊天界面提供搜索入口与结果列表，支持从搜索结果打开目标会话。
- 点击搜索结果后，聊天视图直接定位到命中消息，并对本次命中的关键词做可见高亮。
- 增加验收测试，覆盖跨会话搜索、跨 provider 搜索、结果跳转定位与高亮展示等真实业务场景。

## Capabilities

### New Capabilities
- `chat-history-full-text-search`: 支持对全部聊天记录正文做跨会话全文搜索，并从结果直接定位到命中消息且高亮关键字。

### Modified Capabilities
- None.

## Impact

- 后端影响：需要新增跨会话聊天搜索 API。该 API 负责统一执行历史搜索，内部可优先使用 `ripgrep` 预筛候选命中，再做 Claude/Codex 语义解析与定位信息补全，而不是让浏览器直接发起本地 CLI 查询。
- 前端影响：需要新增聊天搜索入口、结果列表、结果跳转、命中高亮与会话内滚动定位状态。
- 数据与性能影响：首版可按需扫描历史 JSONL，但需要为大历史量场景预留索引或缓存扩展位。
- 验收影响：需要新增 `tests/spec/` 下的 Playwright 验收测试、README 和变更内 `test_cmd.sh`。
