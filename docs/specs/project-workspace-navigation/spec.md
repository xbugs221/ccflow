## Purpose

定义 CCUI 项目工作区的默认入口、项目内导航壳层切换，以及移动端抽屉式导航的稳定行为。

## Requirements

### Requirement: 项目工作区必须以项目主页路由作为默认入口

系统 MUST 为每个项目提供独立的工作区入口路由，并在用户打开该路由时默认显示项目详情页而不是具体会话内容。该页面 MUST 在顶部提供项目切换入口，但不得渲染项目内工作内容左侧导航。

#### Scenario: 打开项目工作区主页
- **WHEN** 用户打开某个项目的工作区路由 `/project/:projectSlug`
- **THEN** 正文区显示该项目的项目详情页
- **AND** 页面顶部显示项目切换入口
- **AND** 左侧不会出现“需求工作流 / 手动会话”工作区导航

### Requirement: 进入工作流或手动会话后必须切换到项目内导航壳层

系统 MUST 在用户从项目主页进入需求工作流详情页或手动会话页时切换到项目内导航壳层。该壳层的左侧导航 MUST 只展示当前项目内的数据，并固定按“需求工作流”在上、“手动会话”在下的分组顺序显示。

#### Scenario: 从项目主页进入工作流详情页
- **WHEN** 用户在项目主页点击某个需求工作流
- **THEN** 路由切换到该项目作用域下的 workflow 详情页
- **AND** 左侧出现仅属于当前项目的工作区导航
- **AND** 导航中“需求工作流”分组位于“手动会话”分组上方
- **AND** 当前工作流在导航中处于选中状态

#### Scenario: 从项目主页进入手动会话页
- **WHEN** 用户在项目主页点击某个手动会话
- **THEN** 路由切换到该项目作用域下的 session 页
- **AND** 左侧出现仅属于当前项目的工作区导航
- **AND** 当前手动会话在导航中处于选中状态
- **AND** 用户可以直接在左侧导航内切换到同项目的其他工作流或会话

### Requirement: 移动端项目内导航必须通过抽屉呈现

系统 MUST 在移动端复用同一套项目作用域路由和双分组导航语义，但项目内导航只能通过抽屉打开，不得在项目主页或具体工作内容页中长期占据正文宽度。

#### Scenario: 移动端打开会话页工作区导航
- **WHEN** 用户在移动端进入某个项目的 workflow 详情页或手动会话页
- **THEN** 页面默认不显示常驻左侧导航
- **AND** 用户可以通过菜单按钮打开工作区导航抽屉
- **AND** 抽屉中显示当前项目的“需求工作流 / 手动会话”两个分组

### Requirement: 左侧项目内导航不得提供排序和新建入口

系统 MUST 将左侧项目内导航限定为当前项目内工作流和手动会话的切换入口，不得在左侧“需求工作流”或“手动会话”分组标题旁提供排序下拉框或新建按钮。

#### Scenario: 左侧导航仅展示分组和条目

- **WHEN** 用户进入某个项目的 workflow 详情页或手动会话页
- **THEN** 左侧项目内导航显示“需求工作流”和“手动会话”分组
- **AND** “需求工作流”分组标题旁没有“工作流排序”下拉框
- **AND** “需求工作流”分组标题旁没有“新建”按钮
- **AND** “手动会话”分组标题旁没有“手动会话排序”下拉框
- **AND** “手动会话”分组标题旁没有“新建”按钮

#### Scenario: 新建和排序入口保留在项目主页

- **WHEN** 用户打开项目主页
- **THEN** 项目主页仍提供新建工作流入口
- **AND** 项目主页仍提供新建手动会话入口
- **AND** 项目主页仍提供工作流排序和手动会话排序控件
