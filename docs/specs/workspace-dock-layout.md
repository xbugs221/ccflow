## 新增需求

### 需求：聊天正文必须始终位于主工作区中心

系统必须把聊天正文作为主工作区中心区域，文件、终端和源代码管理不得再替换聊天正文。

#### 场景：用户点击文件视图

- **当** 用户已经打开一个项目和会话
- **且** 中间区域正在显示聊天正文
- **当** 用户点击 `文件`
- **则** 中间区域必须继续显示聊天正文
- **且** 文件导航必须显示在右侧边栏

#### 场景：用户点击终端视图

- **当** 用户已经打开一个项目和会话
- **且** 中间区域正在显示聊天正文
- **当** 用户点击 `终端`
- **则** 中间区域必须继续显示聊天正文
- **且** 终端必须显示在底部 panel

#### 场景：用户点击源代码管理视图

- **当** 用户已经打开一个项目和会话
- **且** 中间区域正在显示聊天正文
- **当** 用户点击 `源代码管理`
- **则** 中间区域必须继续显示聊天正文
- **且** 源代码管理必须显示在右侧边栏

### 需求：默认布局必须像 IDE dock 一样组织工具面板

系统必须默认把文件导航放在右侧边栏，把终端放在底部 panel。

#### 场景：用户打开项目工作区

- **当** 用户选择一个项目
- **则** 中间区域必须显示聊天或项目主页内容
- **且** 文件导航必须默认显示在右侧边栏
- **且** 终端必须默认显示在底部 panel
- **且** 页面不得把文件导航或终端作为互斥主 tab 替换中间区域

#### 场景：没有选中项目

- **当** 用户没有选中项目
- **则** 中间区域必须显示空状态或项目选择状态
- **且** 文件、终端和源代码管理不得显示可误操作的项目面板

### 需求：文件视图和源代码管理必须在右侧边栏互斥显示

系统必须让文件和源代码管理共享右侧边栏活动区，而不是同时显示两套右侧工具。

#### 场景：从文件切换到源代码管理

- **当** 右侧边栏正在显示文件导航
- **当** 用户点击 `源代码管理`
- **则** 右侧边栏必须显示源代码管理
- **且** 文件导航必须从右侧边栏隐藏
- **且** 中间聊天正文必须保持可见

#### 场景：从源代码管理切换回文件

- **当** 右侧边栏正在显示源代码管理
- **当** 用户点击 `文件`
- **则** 右侧边栏必须显示文件导航
- **且** 源代码管理必须从右侧边栏隐藏
- **且** 中间聊天正文必须保持可见

### 需求：终端必须默认显示在底部 panel

系统必须把终端作为底部 dock panel，而不是主内容 tab。

#### 场景：用户展开终端

- **当** 终端处于隐藏或折叠状态
- **当** 用户点击 `终端`
- **则** 终端必须在底部 panel 展开
- **且** 中间聊天正文必须保持可见

#### 场景：用户折叠终端

- **当** 终端正在底部 panel 展开
- **当** 用户再次点击 `终端` 或点击折叠按钮
- **则** 终端必须折叠或隐藏
- **且** 中间聊天正文必须获得更多可用高度

### 需求：右侧边栏和底部 panel 必须支持折叠和拉伸

用户必须可以控制工具面板尺寸，并能在需要时收起面板。

#### 场景：用户拉伸右侧边栏

- **当** 右侧边栏正在显示文件或源代码管理
- **当** 用户拖动右侧边栏 resize handle
- **则** 右侧边栏宽度必须变化
- **且** 中间聊天正文必须重新占用剩余宽度
- **且** 刷新页面后应恢复用户调整后的宽度

#### 场景：用户拉伸底部终端

- **当** 底部终端正在展开
- **当** 用户拖动底部 panel resize handle
- **则** 底部终端高度必须变化
- **且** 中间聊天正文必须重新占用剩余高度
- **且** 刷新页面后应恢复用户调整后的高度

#### 场景：用户折叠右侧边栏

- **当** 右侧边栏正在展开
- **当** 用户点击折叠按钮
- **则** 右侧边栏必须折叠
- **且** 中间聊天正文必须获得更多可用宽度
- **且** 用户再次展开时必须恢复之前的活动视图和宽度

### 需求：工具面板必须支持全屏

用户必须可以临时把一个工具面板放大全屏查看，并能恢复原布局。

#### 场景：终端全屏和退出

- **当** 终端正在底部 panel 展开
- **当** 用户点击终端全屏按钮
- **则** 终端必须占据主内容区域
- **且** 聊天正文可以暂时隐藏但不得卸载会话状态
- **当** 用户退出全屏
- **则** 终端必须回到底部 panel
- **且** 聊天正文必须恢复可见

#### 场景：右侧面板全屏和退出

- **当** 右侧边栏正在显示文件或源代码管理
- **当** 用户点击全屏按钮
- **则** 当前右侧面板必须占据主内容区域
- **当** 用户退出全屏
- **则** 当前面板必须回到右侧边栏
- **且** 聊天正文必须恢复可见

### 需求：终端必须支持拖拽重排到右侧边栏下半区

系统必须支持把底部终端拖动到右侧边栏下半区，形成右侧上下分区。

#### 场景：用户拖动终端到右侧下半区

- **当** 底部终端正在展开
- **且** 右侧边栏正在显示文件或源代码管理
- **当** 用户拖动终端面板到右侧边栏下半区
- **则** 右侧边栏必须拆分为上下两个区域
- **且** 上方区域必须保留文件或源代码管理
- **且** 下方区域必须显示终端
- **且** 底部 panel 不得继续占用高度
- **且** 中间聊天正文必须保持可见

#### 场景：用户把右侧终端拖回底部

- **当** 终端正在右侧边栏下半区显示
- **当** 用户拖动终端到页面底部 drop 区
- **则** 终端必须回到底部 panel
- **且** 右侧边栏必须恢复为单一文件或源代码管理区域
- **且** 中间聊天正文必须保持可见

### 需求：布局状态必须持久化且可恢复

系统必须记住用户的布局调整，并能处理旧状态或坏状态。

#### 场景：刷新后恢复布局

- **当** 用户调整右侧宽度、底部高度或把终端拖到右侧下半区
- **当** 用户刷新页面
- **则** 页面必须恢复上一次布局
- **且** 聊天正文、文件或源代码管理、终端必须出现在对应位置

#### 场景：存在旧 activeTab 状态

- **当** localStorage 中保存了旧的 `activeTab = files`、`shell` 或 `git`
- **则** 应用启动后不得把中间区域替换成旧 tab 内容
- **且** 必须迁移或忽略旧状态
- **且** 中间区域必须显示聊天正文或项目主页内容

#### 场景：布局持久化数据损坏

- **当** 布局持久化数据无法解析或字段无效
- **则** 应用必须回退默认布局
- **且** 不得显示空白主工作区

### 需求：移动端必须保留可用布局

移动端必须以 overlay 或 sheet 方式展示工具面板，避免把聊天区挤到不可用。

#### 场景：移动端打开文件或源代码管理

- **当** 用户在移动端点击 `文件` 或 `源代码管理`
- **则** 对应面板必须以抽屉或 overlay 方式展示
- **且** 用户关闭面板后必须回到聊天正文

#### 场景：移动端打开终端

- **当** 用户在移动端点击 `终端`
- **则** 终端必须以底部 sheet 或全屏面板方式展示
- **且** 用户关闭终端后必须回到聊天正文
