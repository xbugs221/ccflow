## 新增需求

### 需求：聊天正文必须始终位于主工作区中心

系统必须把聊天正文作为主工作区中心区域，文件、终端和源代码管理不得再替换聊天正文。

#### 场景：用户点击文件视图

- **当** 用户已经打开一个项目和会话
- **且** 中间区域正在显示聊天正文
- **当** 用户点击 `文件`
- **则** 中间区域必须继续显示聊天正文
- **且** 文件导航必须显示在右侧边栏

#### 场景：用户点击终端视图

- **当** 用户已经打开一个项目和会话
- **且** 中间区域正在显示聊天正文
- **当** 用户点击 `终端`
- **则** 中间区域必须继续显示聊天正文
- **且** 终端必须显示在底部 panel

#### 场景：用户点击源代码管理视图

- **当** 用户已经打开一个项目和会话
- **且** 中间区域正在显示聊天正文
- **当** 用户点击 `源代码管理`
- **则** 中间区域必须继续显示聊天正文
- **且** 源代码管理必须显示在右侧边栏

### 需求：默认布局必须像 IDE dock 一样组织工具面板

系统必须把文件导航、终端和源代码管理组织为辅助 dock，但桌面端默认不得展开这些辅助 dock。

#### 场景：用户打开项目工作区

- **当** 用户选择一个项目
- **则** 中间区域必须显示聊天或项目主页内容
- **且** 文件导航不得默认显示在右侧边栏
- **且** 终端不得默认显示在底部 panel
- **且** 页面不得把文件导航或终端作为互斥主 tab 替换中间区域

#### 场景：没有选中项目

- **当** 用户没有选中项目
- **则** 中间区域必须显示空状态或项目选择状态
- **且** 文件、终端和源代码管理不得显示可误操作的项目面板

### 需求：文件视图和源代码管理必须在右侧边栏互斥显示

系统必须让文件和源代码管理共享右侧边栏活动区，而不是同时显示两套右侧工具。

#### 场景：从文件切换到源代码管理

- **当** 右侧边栏正在显示文件导航
- **当** 用户点击 `源代码管理`
- **则** 右侧边栏必须显示源代码管理
- **且** 文件导航必须从右侧边栏隐藏
- **且** 中间聊天正文必须保持可见

#### 场景：从源代码管理切换回文件

- **当** 右侧边栏正在显示源代码管理
- **当** 用户点击 `文件`
- **则** 右侧边栏必须显示文件导航
- **且** 源代码管理必须从右侧边栏隐藏
- **且** 中间聊天正文必须保持可见

### 需求：桌面 dock 工作区必须横向填满主内容区域

桌面端打开项目工作区时，dock 工作区必须占满全局侧边栏之外的主内容宽度，不得按内部 pane 内容宽度收缩。

#### 场景：打开项目会话时工作区填满主内容

- **当** 用户在桌面宽度打开一个项目会话
- **且** 右侧文件或源代码管理 dock 正在显示
- **则** dock 工作区必须横向填满主内容区域
- **且** 右侧 dock 的右边界必须贴近主内容区域右边界
- **且** 右侧 dock 右侧不得出现异常大空白

#### 场景：聊天中心区域占用剩余宽度

- **当** 用户在桌面宽度打开一个项目会话
- **且** 右侧 dock 和底部 terminal dock 正在显示
- **则** 聊天中心区域必须占用右侧 dock 之外的剩余宽度
- **且** 聊天区域不得被压缩成只占页面中部一小段
- **且** 聊天输入框和消息区域必须保持可见可用

### 需求：项目主页分支必须使用相同横向布局

项目主页和具体会话都使用 dock 工作区时，横向填充行为必须一致。

#### 场景：打开项目主页时工作区填满主内容

- **当** 用户在桌面宽度打开项目主页
- **且** 右侧文件或源代码管理 dock 正在显示
- **则** dock 工作区必须横向填满主内容区域
- **且** 项目主页中心区域必须占用右侧 dock 之外的剩余宽度
- **且** 右侧 dock 右侧不得出现异常大空白

### 需求：终端必须默认显示在底部 panel

系统必须把终端作为底部 dock panel，而不是主内容 tab。

#### 场景：用户展开终端

- **当** 终端处于隐藏或折叠状态
- **当** 用户点击 `终端`
- **则** 终端必须在底部 panel 展开
- **且** 中间聊天正文必须保持可见

#### 场景：用户折叠终端

- **当** 终端正在底部 panel 展开
- **当** 用户再次点击顶部 `终端` 工具按钮
- **则** 终端必须折叠或隐藏
- **且** 中间聊天正文必须获得更多可用高度

### 需求：右侧边栏和底部 panel 必须支持折叠和拉伸

用户必须可以控制工具面板尺寸，并能在需要时收起面板。

#### 场景：用户拉伸右侧边栏

- **当** 右侧边栏正在显示文件或源代码管理
- **当** 用户拖动右侧边栏 resize handle
- **则** 右侧边栏宽度必须变化
- **且** 中间聊天正文必须重新占用剩余宽度
- **且** 刷新页面后应恢复用户调整后的宽度

#### 场景：向左拖动右侧边栏边界

- **当** 用户打开项目工作区
- **且** 右侧 dock 正在显示文件或源代码管理
- **当** 用户把右侧 dock 左边界 resize handle 向左拖动
- **则** 右侧 dock 宽度必须增加
- **且** 右侧 dock 的右边界必须保持贴近主内容区域右边界
- **且** 中间聊天区域必须保持可见

#### 场景：向右拖动右侧边栏边界

- **当** 用户打开项目工作区
- **且** 右侧 dock 正在显示文件或源代码管理
- **当** 用户把右侧 dock 左边界 resize handle 向右拖动
- **则** 右侧 dock 宽度必须减少
- **且** 右侧 dock 的右边界必须保持贴近主内容区域右边界
- **且** 右侧 dock 右侧不得因为 resize 出现空白断层
- **且** 宽度不得小于系统允许的最小值

#### 场景：用户拉伸底部终端

- **当** 底部终端正在展开
- **当** 用户拖动底部 panel resize handle
- **则** 底部终端高度必须变化
- **且** 中间聊天正文必须重新占用剩余高度
- **且** 刷新页面后应恢复用户调整后的高度

#### 场景：向上拖动终端上边界

- **当** 用户打开项目工作区
- **且** 底部 terminal dock 正在展开
- **当** 用户把底部 dock 上边界 resize handle 向上拖动
- **则** 底部 terminal dock 高度必须增加
- **且** 中间聊天区域必须保持可见

#### 场景：向下拖动终端上边界

- **当** 用户打开项目工作区
- **且** 底部 terminal dock 正在展开
- **当** 用户把底部 dock 上边界 resize handle 向下拖动
- **则** 底部 terminal dock 高度必须减少
- **且** 高度不得小于系统允许的最小值

#### 场景：用户折叠右侧边栏

- **当** 右侧边栏正在展开
- **当** 用户再次点击顶部当前 active 的 `文件` 或 `源代码管理` 工具按钮
- **则** 右侧边栏必须折叠
- **且** 中间聊天正文必须获得更多可用宽度
- **且** 用户再次展开时必须恢复之前的活动视图和宽度

### 需求：源代码管理不得撑开右侧 dock

系统必须把源代码管理作为右侧 dock 内部内容显示，不得改变用户已经设置的右侧 dock 尺寸。

#### 场景：从文件切换到源代码管理时保持宽度

- **当** 用户打开项目工作区
- **且** 右侧 dock 正在显示文件
- **且** 用户已经调整过右侧 dock 宽度
- **当** 用户点击源代码管理
- **则** 右侧 dock 必须显示 Git 面板
- **且** 右侧 dock 宽度必须保持用户调整后的宽度
- **且** 中间聊天区域必须保持可见

#### 场景：反复切换右侧工具时不持续向左推进

- **当** 用户打开项目工作区
- **且** 右侧 dock 正在显示文件或源代码管理
- **当** 用户多次点击文件和源代码管理
- **则** 右侧 dock 不得持续向左扩大
- **且** 右侧 dock 的左边界不得因 Git 内容溢出而持续侵入聊天区域

### 需求：工具面板必须支持全屏

用户必须可以临时把一个工具面板放大全屏查看，并能恢复原布局。

#### 场景：终端全屏和退出

- **当** 终端正在底部 panel 展开
- **当** 用户点击终端全屏按钮
- **则** 终端必须占据主内容区域
- **且** 聊天正文可以暂时隐藏但不得卸载会话状态
- **当** 用户退出全屏
- **则** 终端必须回到底部 panel
- **且** 聊天正文必须恢复可见

#### 场景：终端全屏入口仍然可用

- **当** 用户打开项目工作区
- **且** 底部 terminal dock 正在展开
- **则** 用户仍然可以通过终端 pane 顶部控制条进入终端全屏
- **且** 退出全屏后终端必须回到底部 dock

#### 场景：右侧面板全屏和退出

- **当** 右侧边栏正在显示文件或源代码管理
- **当** 用户点击全屏按钮
- **则** 当前右侧面板必须占据主内容区域
- **当** 用户退出全屏
- **则** 当前面板必须回到右侧边栏
- **且** 聊天正文必须恢复可见

### 需求：终端必须支持拖拽重排到右侧边栏下半区

系统必须支持把底部终端拖动到右侧边栏下半区，形成右侧上下分区。

#### 场景：用户拖动终端到右侧下半区

- **当** 底部终端正在展开
- **且** 右侧边栏正在显示文件或源代码管理
- **当** 用户拖动终端面板到右侧边栏下半区
- **则** 右侧边栏必须拆分为上下两个区域
- **且** 上方区域必须保留文件或源代码管理
- **且** 下方区域必须显示终端
- **且** 底部 panel 不得继续占用高度
- **且** 中间聊天正文必须保持可见

#### 场景：用户把右侧终端拖回底部

- **当** 终端正在右侧边栏下半区显示
- **当** 用户拖动终端到页面底部 drop 区
- **则** 终端必须回到底部 panel
- **且** 右侧边栏必须恢复为单一文件或源代码管理区域
- **且** 中间聊天正文必须保持可见

### 需求：布局状态必须持久化且可恢复

系统必须记住用户的布局调整，并能处理旧状态或坏状态。

#### 场景：刷新后恢复布局

- **当** 用户调整右侧宽度、底部高度或把终端拖到右侧下半区
- **当** 用户刷新页面
- **则** 页面必须恢复上一次布局
- **且** 聊天正文、文件或源代码管理、终端必须出现在对应位置

#### 场景：存在旧 activeTab 状态

- **当** localStorage 中保存了旧的 `activeTab = files`、`shell` 或 `git`
- **则** 应用启动后不得把中间区域替换成旧 tab 内容
- **且** 必须迁移或忽略旧状态
- **且** 中间区域必须显示聊天正文或项目主页内容

#### 场景：布局持久化数据损坏

- **当** 布局持久化数据无法解析或字段无效
- **则** 应用必须回退到只显示中心会话或项目主页的默认布局
- **且** 不得显示空白主工作区

### 需求：桌面工作区默认不得展开辅助 dock

桌面端进入项目或会话页时，默认必须聚焦会话主区，文件、终端、Git dock 只能由用户主动打开。

#### 场景：打开桌面会话页

- **给定** 用户打开一个项目会话路由
- **当** 页面完成加载
- **则** 会话主区必须可见
- **且** 底部输入框必须可见
- **且** 右侧文件 dock 不应默认可见
- **且** 底部终端 dock 不应默认可见

#### 场景：桌面主动打开辅助 dock

- **给定** 用户正在查看桌面会话页
- **当** 用户点击 `文件`、`终端` 或 `源代码管理`
- **则** 对应 dock 必须打开
- **且** 会话 transcript 必须仍然可见
- **且** 底部输入框必须仍然可见

### 需求：移动端必须保留可用布局

移动端必须以 overlay 或 sheet 方式展示工具面板，避免把聊天区挤到不可用。

#### 场景：移动端打开文件或源代码管理

- **当** 用户在移动端点击 `文件` 或 `源代码管理`
- **则** 对应面板必须以抽屉或 overlay 方式展示
- **且** 用户关闭面板后必须回到聊天正文

#### 场景：移动端打开终端

- **当** 用户在移动端点击 `终端`
- **则** 终端必须以底部 sheet 或全屏面板方式展示
- **且** 用户关闭终端后必须回到聊天正文

### 需求：主会话区域必须可以上下滚动

当聊天消息内容超过可视高度时，用户必须能在主会话区域查看上方和下方消息。

#### 场景：长会话消息列表滚动

- **当** 用户打开项目并进入一个包含长消息内容的会话
- **且** 消息内容高度超过主会话可视区域
- **当** 用户在消息区域滚轮向下或拖动滚动条
- **则** 消息区域的滚动位置必须变化
- **且** 用户必须能看到滚动后的消息内容
- **且** 输入框必须保持可用，不得因消息滚动而消失到不可操作位置

#### 场景：终端展开后聊天仍可滚动

- **当** 用户打开项目会话
- **且** 底部终端 pane 正在展开并占用一部分高度
- **当** 用户在主会话消息区域滚动
- **则** 主会话区域仍必须可以上下滚动
- **且** 终端不得遮挡消息区域的滚动内容

### 需求：项目主页必须可以上下滚动

当项目主页内容超过可视高度时，用户必须能滚动查看完整主页内容。

#### 场景：项目主页内容溢出后滚动

- **当** 用户打开一个项目主页
- **且** 项目主页中的会话、工作流或概览内容超过中心区域高度
- **当** 用户在项目主页中心区域滚轮向下或拖动滚动条
- **则** 项目主页的滚动位置必须变化
- **且** 用户必须能看到滚动后的主页内容
- **且** 顶部工作区 tab 和 dock 外框不得跟随主页内容滚走

#### 场景：项目主页在 dock 面板展开时仍可滚动

- **当** 用户打开项目主页
- **且** 右侧文件或源代码管理 pane 展开
- **且** 底部终端 pane 展开
- **当** 用户在项目主页中心区域滚动
- **则** 项目主页必须仍可上下滚动
- **且** 右侧和底部 pane 不得覆盖主页内容

### 需求：顶部工作区 tab 必须只显示 icon

`消息`、`终端`、`文件`、`源代码管理` 顶部 tab 必须以 icon 工具栏形式显示，不再显示可见文字。

#### 场景：顶部 tab 不显示文字

- **当** 用户打开项目工作区
- **则** 顶部工作区 tab 必须显示 `消息`、`终端`、`文件`、`源代码管理` 对应 icon
- **且** tab 按钮内不得显示 `消息`、`终端`、`文件`、`源代码管理` 可见文字
- **且** tab 的点击区域必须保持稳定，不得因文字隐藏造成布局跳动

#### 场景：icon-only tab 保留可访问名称

- **当** 用户使用辅助技术或鼠标悬停查看顶部 tab
- **则** 每个 icon-only tab 仍必须提供对应名称
- **且** 自动化测试仍可以通过稳定的 `data-testid` 和选中态判断当前 tab 状态

### 需求：pane 控制按钮必须位于顶部

终端、文件、源代码管理等 dock pane 的移动和全屏按钮必须位于 pane 顶部控制条。

#### 场景：终端 pane 顶部显示移动和全屏按钮

- **当** 用户打开项目工作区
- **且** 底部终端 pane 正在展开
- **则** 终端 pane 的移动按钮必须显示在 pane 顶部控制条
- **且** 终端 pane 的全屏按钮必须显示在 pane 顶部控制条
- **且** 终端 pane 底部不得再显示这些控制按钮

#### 场景：右侧 pane 顶部显示全屏按钮

- **当** 用户打开项目工作区
- **且** 右侧 pane 正在显示文件或源代码管理
- **则** 当前 pane 的全屏按钮必须显示在 pane 顶部控制条
- **且** pane 底部不得再显示这些控制按钮

### 需求：pane 内部不得再提供折叠按钮

折叠和展开必须统一通过顶部对应 tab 完成，pane 内部不得再提供重复折叠入口。

#### 场景：点击终端 tab 折叠和展开终端

- **当** 底部终端 pane 正在展开
- **当** 用户点击顶部 `终端` icon tab
- **则** 底部终端 pane 必须折叠
- **当** 用户再次点击顶部 `终端` icon tab
- **则** 底部终端 pane 必须重新展开

#### 场景：点击文件或源代码管理 tab 折叠和展开右侧 pane

- **当** 右侧 pane 正在显示文件或源代码管理
- **当** 用户点击顶部当前 active 的 icon tab
- **则** 右侧 pane 必须折叠
- **当** 用户再次点击对应 icon tab
- **则** 右侧 pane 必须重新展开并恢复对应内容

#### 场景：pane 内不显示折叠按钮

- **当** 用户打开终端、文件或源代码管理 pane
- **则** pane 顶部或底部控制条中不得出现独立折叠按钮
- **且** 既有移动和全屏能力必须保持可用

### 需求：移动端必须使用单视图工作区

移动端必须一次只展示一个主视图，不得继续使用桌面 dock、pane 或 overlay 模型展示工具区。

#### 场景：移动端打开会话默认显示聊天

- **当** 用户在移动端打开一个项目会话
- **则** 页面必须显示聊天主视图
- **且** 页面不得显示右侧 dock
- **且** 页面不得显示底部 dock
- **且** 页面不得显示 dock resize handle
- **且** 页面不得显示移动端 overlay close 按钮

#### 场景：移动端切换到文件视图

- **当** 用户在移动端打开一个项目会话
- **当** 用户点击 `文件`
- **则** 页面必须切换为整屏文件视图
- **且** 聊天主视图必须从当前主区域隐藏
- **且** 页面不得显示右侧 dock 或 overlay close 按钮

#### 场景：移动端切换到终端视图

- **当** 用户在移动端打开一个项目会话
- **当** 用户点击 `终端`
- **则** 页面必须切换为整屏终端视图
- **且** 终端不得作为底部 dock 显示
- **且** 页面不得显示 terminal move 控制

#### 场景：移动端切换到源代码管理视图

- **当** 用户在移动端打开一个项目会话
- **当** 用户点击 `源代码管理`
- **则** 页面必须切换为整屏源代码管理视图
- **且** 文件视图和聊天主视图必须从当前主区域隐藏
- **且** 页面不得显示右侧 dock 或 overlay close 按钮

#### 场景：移动端从工具视图回到聊天

- **当** 用户在移动端处于文件、源代码管理或终端视图
- **当** 用户点击 `聊天`
- **则** 页面必须切换回整屏聊天主视图
- **且** 不需要点击额外关闭按钮

### 需求：移动端会话页必须可以上下滚动

移动端长会话必须能在会话内容区内滚动，不能因为外层布局锁死而无法浏览历史。

#### 场景：移动端浏览长会话

- **给定** 一个包含多屏历史消息的项目会话
- **当** 用户在移动端打开该会话
- **则** 会话 transcript 的内容高度必须大于可见高度
- **当** 用户向上或向下滚动会话区
- **则** 会话区滚动位置必须随操作变化
- **且** 页面不得跳到空白主区

### 需求：移动端底部输入框必须保持可见

移动端聊天输入框是继续对话的核心入口，不能被父容器裁切或推到视口外。

#### 场景：打开移动端已有会话

- **给定** 用户使用移动端视口
- **当** 用户打开一个已有项目会话
- **则** 底部输入框必须在当前视口内可见
- **且** 输入框必须可编辑
- **且** 会话内容滚动不得遮挡输入框

#### 场景：移动端切换回会话单视图

- **给定** 用户在移动端从会话页切到文件、终端或源代码管理单视图
- **当** 用户切回 `会话`
- **则** 会话 transcript 必须恢复可见
- **且** 底部输入框必须恢复可见

### 需求：打开已有会话不得显示纯空白

已有会话点开后必须呈现历史消息、明确空状态或错误状态，不能静默显示空白。

#### 场景：打开有历史消息的会话

- **给定** 一个后端能返回历史消息的会话
- **当** 用户点开该会话
- **则** 页面必须显示该会话的历史消息
- **且** 不得显示纯空白主区

#### 场景：打开真实空历史会话

- **给定** 一个已创建但尚无历史消息的会话
- **当** 用户点开该会话
- **则** 页面必须显示明确的空历史或继续输入状态
- **且** 底部输入框必须可见
- **且** 不得显示纯空白主区

#### 场景：消息加载失败

- **给定** 会话消息接口返回失败
- **当** 用户打开该会话
- **则** 页面必须显示明确的加载失败状态
- **且** 不得把失败静默伪装成空白会话
- **且** 用户仍能看到返回、刷新或继续操作入口中的至少一种

### 需求：移动端会话路由必须解析到正确消息来源

移动端和桌面端打开同一会话路由时，应加载同一份 provider 历史消息。

#### 场景：打开 cN 路由绑定的 provider 会话

- **给定** `cN` 路由已经绑定真实 provider session id
- **当** 用户从移动端打开该 `cN` 会话路由
- **则** 前端必须请求并显示真实 provider 会话历史
- **且** 不得因为只使用 route id 而得到空历史

#### 场景：打开尚未绑定 provider 的草稿会话

- **给定** `cN` 路由尚未发送首条消息，也没有 provider session id
- **当** 用户打开该草稿会话
- **则** 页面必须显示新会话或继续输入状态
- **且** 不得误报为历史加载失败

### 需求：移动端文件打开必须保持文件语义

移动端文件视图中打开文件后，必须保持在文件/编辑器上下文内，而不是自动退回聊天。

#### 场景：移动端从文件树打开文件

- **当** 用户在移动端处于文件视图
- **当** 用户打开一个文件
- **则** 页面必须显示移动端文件编辑器
- **且** 当前主视图仍属于文件/编辑器上下文
- **且** 页面不得自动切回聊天主视图

### 需求：桌面端 dock 布局不得回退

桌面端必须继续保留现有工作区 dock 能力。

#### 场景：桌面端打开项目会话

- **当** 用户在桌面端打开一个项目会话
- **则** 页面必须继续显示中心聊天区域
- **且** 文件或源代码管理必须可通过顶部工具按钮显示在右侧 dock
- **且** 终端必须可通过顶部工具按钮显示在底部 dock

#### 场景：桌面端使用 dock 控制

- **当** 用户在桌面端点击 `文件`、`源代码管理` 或 `终端`
- **则** 这些按钮必须继续控制 dock 展开、折叠或切换
- **且** 不得退化为移动端的整屏互斥视图

### 需求：顶部会话恢复信息必须只显示恢复 id

会话标题下方的恢复信息必须只显示恢复会话所需的 id，不得显示 provider CLI 命令前缀或参数。

#### 场景：Codex 会话显示恢复 id

- **当** 用户打开带有 Codex `providerSessionId` 的会话
- **则** 顶部恢复信息必须只显示该 id
- **且** 不得包含 `codex`
- **且** 不得包含 `resume`
- **且** 不得包含 `--dangerously-bypass-approvals-and-sandbox`

#### 场景：OpenCode 会话显示恢复 id

- **当** 用户打开带有 OpenCode `providerSessionId` 的会话
- **则** 顶部恢复信息必须只显示该 id
- **且** 不得包含 `opencode`
- **且** 不得包含 `--session`

#### 场景：不可恢复会话不显示恢复信息

- **当** 用户打开临时会话、ccflow 路由别名会话或没有真实 provider resume id 的会话
- **则** 顶部不得显示恢复信息行

### 需求：桌面工作区默认必须只选中会话 tab

进入项目或会话后，桌面主工作区必须以会话为默认主内容，文件、终端和源代码管理只能作为辅助 dock 状态。

#### 场景：打开项目会话路由

- **给定** 用户本地没有有效的工作区 tab 状态
- **当** 用户打开一个项目会话路由
- **则** `会话` tab 必须处于选中状态
- **且** `文件`、`终端`、`源代码管理` tab 不得表现为主选中状态
- **且** 会话区必须显示项目会话内容或会话加载状态

#### 场景：旧 activeTab 保存为辅助 dock

- **给定** 浏览器本地保存的旧 `activeTab` 是 `files`、`shell` 或 `git`
- **当** 用户重新打开项目或会话页面
- **则** 主内容必须回到 `会话`
- **且** 旧值最多只能影响对应 dock 是否展开
- **且** 页面不得默认进入空白会话区

### 需求：普通前端页面不得自动刷新

普通项目页和会话页不得周期性重载页面或刷新项目状态导致用户界面跳动。

#### 场景：普通会话页保持稳定

- **给定** 用户正在查看普通项目会话
- **当** 用户不进行任何操作并等待数秒
- **则** 页面不得发生整页 reload
- **且** 会话区不得被清空或重置
- **且** 前端不得每秒请求项目列表来刷新普通会话页

#### 场景：运行中 workflow 仍可刷新状态

- **给定** 用户正在查看运行中的 workflow 详情
- **当** 后端 workflow 状态仍在变化
- **则** 前端可以刷新 workflow 状态
- **但** 该刷新不得影响普通会话页
- **且** 该刷新不得触发整页 reload

### 需求：底部终端栏不得显示断开和重启控制行

底部终端 dock 应只显示 dock 面板控制和终端内容，不重复显示 Shell 自己的连接控制栏。

#### 场景：打开底部终端 dock

- **给定** 用户已经打开项目工作区
- **当** 底部终端 dock 可见
- **则** 终端 dock 内不得显示 `断开连接`、`断开链接` 或 `重启` 控制行
- **且** 终端画布必须仍然可见
- **且** dock 面板自身的标题、移动和全屏控制可以保留

### 需求：终端栏必须支持新建和删除终端

用户可以在当前项目下创建多个独立终端，并关闭当前不需要的终端。

#### 场景：新建一个终端

- **给定** 用户已经打开底部终端 dock 或右侧 split 中的终端区
- **当** 用户点击 `新建`
- **则** 终端栏必须新增一个终端实例
- **且** 新终端必须成为当前活动终端
- **且** 原会话区必须仍然可见

#### 场景：删除当前终端

- **给定** 用户已经打开至少一个终端实例
- **当** 用户点击 `删除`
- **则** 当前终端实例必须关闭
- **且** 前端必须关闭该终端对应的 shell 连接
- **且** 删除终端不得删除项目、会话、文件或历史记录
- **且** 原会话区必须仍然可见

#### 场景：删除最后一个终端后仍可恢复

- **给定** 用户只剩一个终端实例
- **当** 用户删除该终端
- **则** 页面必须保留可继续新建终端的入口
- **且** 工作区不得崩溃或清空会话区

### 需求：文件 tab 内部不得重复显示“文件”标题

文件 dock 外层已经提供标题时，内部文件树不应再重复显示同样文字。

#### 场景：打开文件 dock

- **给定** 用户打开项目工作区
- **当** 文件 dock 可见
- **则** dock 外层可以显示 `文件`
- **但** 文件树内部工具栏不得再次显示独立的 `文件` 标题
- **且** 搜索、刷新、新建文件、新建文件夹、上传、折叠等文件操作必须仍可使用
