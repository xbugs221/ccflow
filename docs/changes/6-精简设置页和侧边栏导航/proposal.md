## 背景

ccflow 的设置页继承了过多历史功能，当前已经和产品职责不匹配：

- `Git` tab 会读取和写入全局 git identity，容易误导用户以为 ccflow 要求统一配置全局 git。
- `API 和令牌` tab、外部 API 文档和 API key 管理不属于当前 ccflow 的核心交互。
- `外观` tab 中的项目排序和代码编辑器设置不是当前用户完成工作流的必要设置。
- `智能体` tab 里还保留 MCP 服务器卡片，但 MCP 配置不应在这次设置页主流程里暴露。
- OpenCode 卡片现在复用 Codex logo，且没有把 OpenCode 内部已经连接的 provider 状态展示出来。
- `诊断` tab 内容仍是英文，中文界面下不可读。
- 日语和韩语 locale 仍被打包和暴露，但当前维护资源集中在中文和英文。
- 左侧导航栏顶部堆叠了多个工具按钮，其中“搜索项目”与当前主要工作流关系不大，且顶部按钮挤占项目导航的首屏注意力。

这次变更目标是把设置页收敛到真实需要用户处理的事项：显示偏好、智能体连接状态、运行诊断。

```text
当前设置页
  |
  +-- 外观
  |     +-- 深色模式
  |     +-- 语言
  |     +-- 项目排序            [移除]
  |     +-- 代码编辑器          [移除]
  |
  +-- Git                     [移除]
  +-- API 和令牌              [移除]
  |
  +-- 智能体
  |     +-- 账户
  |     +-- MCP 服务器         [移除]
  |
  +-- 诊断                    [中文化]

左侧导航栏
  |
  +-- 顶部：品牌/标题
  |
  +-- 项目列表
  |
  +-- 底部：刷新 / 新建项目 / 搜索聊天记录 / 设置 / 收起侧栏
        搜索项目 [移除]
```

## 变更内容

- 设置页顶层只保留：
  - `外观`
  - `智能体`
  - `诊断`
- `外观` tab 只保留深色模式和语言选择，移除项目排序和代码编辑器设置组。
- 移除设置页 `Git` tab，并删除对应的前端源码、全局 git config 后端入口、测试和文档。
- 移除设置页 `API 和令牌` tab，并删除对应的设置页源码、API key 管理入口、测试和文档；若项目创建仍需要一次性 GitHub token，应保留在项目创建流程内，不再作为设置页持久配置入口。
- `智能体` tab 只保留账户/连接状态视图，移除 `MCP 服务器` 子 tab 和卡片。
- `OpenCode` 在设置页、会话入口等 provider logo 位置使用 OpenCode 自己的 icon，不再落到 Codex/ChatGPT logo。
- 后端暴露 OpenCode 连接状态，前端展示 OpenCode 内部已连接的 provider，例如已连接的 provider 名称、可用状态和来源摘要。
- `诊断` tab 的标题、说明、状态、字段名、加载态和错误态在中文界面下全部显示中文。
- i18n 只保留 `en` 和 `zh-CN`，移除 `ja`、`ko` 的语言配置、locale 文件引用、语言选择项、测试引用和文档说明。
- 左侧导航栏移除“搜索项目”按钮和展开式项目搜索输入，其余工具按钮从顶部移动到底部工具区。

## 能力范围

```text
settings
  |
  +-- appearance
  |     +-- theme
  |     +-- language: en / zh-CN
  |
  +-- agents
  |     +-- codex account status
  |     +-- opencode provider icon
  |     +-- opencode connected provider status
  |
  +-- diagnostics
        +-- localized runtime dependency status
```

## 非目标

- 不移除项目工作区里的 Git 面板、diff、提交历史或分支操作。
- 不改变 `.wo/runs/` 运行态协议，也不创建任何 `.wo/runs/` 文件。
- 不实现新的 MCP 管理能力；本次只从设置页移除 MCP 服务器卡片。
- 不重做 OpenCode 聊天执行协议；只补齐 icon 和连接状态展示。
- 不引入新的语言；本次只保留中文和英文。
- 不移除全局聊天记录搜索；只移除左侧导航里的项目搜索过滤入口。

## 开放问题

无阻塞开放问题。执行阶段需要先做调用点审计，确认 API key 和 GitHub token 管理入口删除后不会破坏项目创建的一次性 token 流程。
