## 新增需求

### 需求：工作流卡片必须聚合多轮审核和修复阶段

项目概览和侧边栏里的工作流进度不应因为多轮审核/修复不断增加图标数量。

#### 场景：工作流已经经历两轮审核和一轮修复

- **给定** 一个工作流包含 `execution`、`review_1`、`repair_1`、`review_2`
- **当** 用户查看项目概览工作流卡片
- **则** 卡片必须只显示一个审核图标
- **且** 审核图标旁必须显示 `x2`
- **且** 卡片必须只显示一个修复图标
- **且** 修复图标旁必须显示 `x1`

#### 场景：聚合阶段中有正在运行或失败的轮次

- **给定** 一个工作流包含多个审核阶段
- **且** 其中一个审核阶段处于 `active`、`running`、`blocked` 或 `failed`
- **当** 用户查看工作流卡片
- **则** 审核聚合图标必须显示为活跃状态颜色
- **且** 不得因为其他审核轮次已完成而显示成全部完成状态

### 需求：详情页角色行必须用数字表示完成次数

`写`、`审`、`存` 等角色行不应再用一串重复勾号表示次数。

#### 场景：写入角色已经完成三次

- **给定** 工作流详情页的 `写` 行 `checkCount = 3`
- **当** 用户查看该行
- **则** 页面必须显示数字计数 `x3`
- **且** 页面不得显示 `✓✓✓`

#### 场景：审核角色已经完成两次

- **给定** 工作流详情页的 `审` 行 `checkCount = 2`
- **当** 用户查看该行
- **则** 页面必须显示数字计数 `x2`
- **且** 页面不得显示 `✓✓`

### 需求：详情页角色行会话链接必须短文本跳转

角色行里通往 child session 的链接不能暴露完整 session 编号。

#### 场景：写入角色有可跳转的 child session

- **给定** 工作流详情页的 `写` 行存在 `sessionRef.sessionId = executor-session-long-id`
- **当** 用户查看该行
- **则** 可见链接文本必须是 `会话`
- **且** 页面正文不得显示 `executor-session-long-id`
- **当** 用户点击 `会话`
- **则** 页面必须跳转到原来的 `executor-session-long-id` 会话

#### 场景：审核角色有可跳转的 child session

- **给定** 工作流详情页的 `审` 行存在 `sessionRef.sessionId = reviewer-session-long-id`
- **当** 用户查看该行
- **则** 可见链接文本必须是 `会话`
- **且** 点击后必须进入审核 child session

### 需求：详情页角色行必须显示当前轮次产物链接

用户应在角色行直接打开当前轮次产物，历史轮次产物不应在该位置出现。

#### 场景：第二轮审核已有产物

- **给定** 工作流有 `review-1.json` 和 `review-2.json`
- **且** `审` 行当前计数为 `x2`
- **当** 用户查看工作流详情页
- **则** `审` 行必须在 `会话` 链接后显示 `review-2.json`
- **且** `审` 行不得显示 `review-1.json`
- **当** 用户点击 `review-2.json`
- **则** 页面必须打开第二轮审核产物文件

#### 场景：当前轮次产物尚未生成

- **给定** 角色行存在 `会话` 链接
- **但** 当前轮次产物不存在或 `exists = false`
- **当** 用户查看工作流详情页
- **则** 不得显示一个会跳转失败的产物链接
- **且** 会话链接仍必须可用
