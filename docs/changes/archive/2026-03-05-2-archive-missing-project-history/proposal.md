## Why

当前 WebUI 左侧项目与对话历史未校验项目路径是否仍然存在，导致项目被删除或移动后仍持续展示，用户点击后无法使用，产生噪音和误导。需要在页面刷新时自动识别失效项目并归档其历史，保证列表只展示可继续操作的项目。

## What Changes

- 在页面刷新触发的数据加载流程中增加一次项目路径存在性检测。
- 当检测到项目路径不存在时，将该项目及其关联会话历史标记为已归档，而不是继续参与活跃列表。
- `/api/projects` 仅返回未归档且路径有效的项目，避免 WebUI 左侧出现不可用项。
- 为归档项目保留历史数据，便于后续审计或恢复，不做物理删除。

## Capabilities

### New Capabilities
- `project-history-cleanup-on-missing-path`: 在刷新时校验项目路径有效性，并自动归档失效项目及其历史，确保 UI 只展示可用项目。

### Modified Capabilities
- None.

## Impact

- 受影响模块：项目聚合与列表接口（`/api/projects`）、项目/会话历史存储读写逻辑、前端刷新后的左侧项目展示。
- 受影响数据：项目历史状态需要新增或复用“归档”标记；现有历史保留但从活跃视图隐藏。
- 风险点：路径检测开销、网络路径/权限异常时的误判、历史状态迁移对旧数据的兼容。
