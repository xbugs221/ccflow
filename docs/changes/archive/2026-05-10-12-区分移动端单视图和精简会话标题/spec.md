## 新增需求

### 需求：移动端必须使用单视图工作区

移动端必须一次只展示一个主视图，不得继续使用桌面 dock、pane 或 overlay 模型展示工具区。

#### 场景：移动端打开会话默认显示聊天

- **当** 用户在移动端打开一个项目会话
- **则** 页面必须显示聊天主视图
- **且** 页面不得显示右侧 dock
- **且** 页面不得显示底部 dock
- **且** 页面不得显示 dock resize handle
- **且** 页面不得显示移动端 overlay close 按钮

#### 场景：移动端切换到文件视图

- **当** 用户在移动端打开一个项目会话
- **当** 用户点击 `文件`
- **则** 页面必须切换为整屏文件视图
- **且** 聊天主视图必须从当前主区域隐藏
- **且** 页面不得显示右侧 dock 或 overlay close 按钮

#### 场景：移动端切换到终端视图

- **当** 用户在移动端打开一个项目会话
- **当** 用户点击 `终端`
- **则** 页面必须切换为整屏终端视图
- **且** 终端不得作为底部 dock 显示
- **且** 页面不得显示 terminal move 控制

#### 场景：移动端切换到源代码管理视图

- **当** 用户在移动端打开一个项目会话
- **当** 用户点击 `源代码管理`
- **则** 页面必须切换为整屏源代码管理视图
- **且** 文件视图和聊天主视图必须从当前主区域隐藏
- **且** 页面不得显示右侧 dock 或 overlay close 按钮

#### 场景：移动端从工具视图回到聊天

- **当** 用户在移动端处于文件、源代码管理或终端视图
- **当** 用户点击 `聊天`
- **则** 页面必须切换回整屏聊天主视图
- **且** 不需要点击额外关闭按钮

### 需求：移动端文件打开必须保持文件语义

移动端文件视图中打开文件后，必须保持在文件/编辑器上下文内，而不是自动退回聊天。

#### 场景：移动端从文件树打开文件

- **当** 用户在移动端处于文件视图
- **当** 用户打开一个文件
- **则** 页面必须显示移动端文件编辑器
- **且** 当前主视图仍属于文件/编辑器上下文
- **且** 页面不得自动切回聊天主视图

### 需求：桌面端 dock 布局不得回退

桌面端必须继续保留现有工作区 dock 能力。

#### 场景：桌面端打开项目会话

- **当** 用户在桌面端打开一个项目会话
- **则** 页面必须继续显示中心聊天区域
- **且** 文件或源代码管理必须继续显示在右侧 dock
- **且** 终端必须继续显示在底部 dock

#### 场景：桌面端使用 dock 控制

- **当** 用户在桌面端点击 `文件`、`源代码管理` 或 `终端`
- **则** 这些按钮必须继续控制 dock 展开、折叠或切换
- **且** 不得退化为移动端的整屏互斥视图

### 需求：顶部会话恢复信息必须只显示恢复 id

会话标题下方的恢复信息必须只显示恢复会话所需的 id，不得显示 provider CLI 命令前缀或参数。

#### 场景：Codex 会话显示恢复 id

- **当** 用户打开带有 Codex `providerSessionId` 的会话
- **则** 顶部恢复信息必须只显示该 id
- **且** 不得包含 `codex`
- **且** 不得包含 `resume`
- **且** 不得包含 `--dangerously-bypass-approvals-and-sandbox`

#### 场景：OpenCode 会话显示恢复 id

- **当** 用户打开带有 OpenCode `providerSessionId` 的会话
- **则** 顶部恢复信息必须只显示该 id
- **且** 不得包含 `opencode`
- **且** 不得包含 `--session`

#### 场景：不可恢复会话不显示恢复信息

- **当** 用户打开临时会话、ccflow 路由别名会话或没有真实 provider resume id 的会话
- **则** 顶部不得显示恢复信息行
