## 新增需求

### 需求：桌面 dock 工作区必须横向填满主内容区域

桌面端打开项目工作区时，dock 工作区必须占满全局侧边栏之外的主内容宽度，不得按内部 pane 内容宽度收缩。

#### 场景：打开项目会话时工作区填满主内容

- **当** 用户在桌面宽度打开一个项目会话
- **且** 右侧文件或源代码管理 dock 正在显示
- **则** dock 工作区必须横向填满主内容区域
- **且** 右侧 dock 的右边界必须贴近主内容区域右边界
- **且** 右侧 dock 右侧不得出现异常大空白

#### 场景：聊天中心区域占用剩余宽度

- **当** 用户在桌面宽度打开一个项目会话
- **且** 右侧 dock 和底部 terminal dock 正在显示
- **则** 聊天中心区域必须占用右侧 dock 之外的剩余宽度
- **且** 聊天区域不得被压缩成只占页面中部一小段
- **且** 聊天输入框和消息区域必须保持可见可用

### 需求：右侧 dock resize 后必须保持右边界稳定

用户调整右侧 dock 宽度时，右侧 dock 必须仍然挂在主内容右边界，宽度变化只能改变 dock 左边界和中心区域宽度。

#### 场景：向左拖动右侧 dock 左边界

- **当** 用户打开桌面项目工作区
- **且** 右侧 dock 正在显示文件或源代码管理
- **当** 用户把右侧 dock 左边界 resize handle 向左拖动
- **则** 右侧 dock 宽度必须增加
- **且** 右侧 dock 的右边界必须保持贴近主内容区域右边界
- **且** 中间聊天或项目主页区域必须随剩余宽度收缩但保持可见

#### 场景：向右拖动右侧 dock 左边界

- **当** 用户打开桌面项目工作区
- **且** 右侧 dock 正在显示文件或源代码管理
- **当** 用户把右侧 dock 左边界 resize handle 向右拖动
- **则** 右侧 dock 宽度必须减少
- **且** 右侧 dock 的右边界必须保持贴近主内容区域右边界
- **且** 右侧 dock 右侧不得因为 resize 出现空白断层

### 需求：项目主页分支也必须使用相同横向布局

项目主页和具体会话都使用 dock 工作区时，横向填充行为必须一致。

#### 场景：打开项目主页时工作区填满主内容

- **当** 用户在桌面宽度打开项目主页
- **且** 右侧文件或源代码管理 dock 正在显示
- **则** dock 工作区必须横向填满主内容区域
- **且** 项目主页中心区域必须占用右侧 dock 之外的剩余宽度
- **且** 右侧 dock 右侧不得出现异常大空白
