## 新增需求

### 需求：主会话区域必须可以上下滚动

当聊天消息内容超过可视高度时，用户必须能在主会话区域查看上方和下方消息。

#### 场景：长会话消息列表滚动

- **当** 用户打开项目并进入一个包含长消息内容的会话
- **且** 消息内容高度超过主会话可视区域
- **当** 用户在消息区域滚轮向下或拖动滚动条
- **则** 消息区域的滚动位置必须变化
- **且** 用户必须能看到滚动后的消息内容
- **且** 输入框必须保持可用，不得因消息滚动而消失到不可操作位置

#### 场景：终端展开后聊天仍可滚动

- **当** 用户打开项目会话
- **且** 底部终端 pane 正在展开并占用一部分高度
- **当** 用户在主会话消息区域滚动
- **则** 主会话区域仍必须可以上下滚动
- **且** 终端不得遮挡消息区域的滚动内容

### 需求：项目主页必须可以上下滚动

当项目主页内容超过可视高度时，用户必须能滚动查看完整主页内容。

#### 场景：项目主页内容溢出后滚动

- **当** 用户打开一个项目主页
- **且** 项目主页中的会话、工作流或概览内容超过中心区域高度
- **当** 用户在项目主页中心区域滚轮向下或拖动滚动条
- **则** 项目主页的滚动位置必须变化
- **且** 用户必须能看到滚动后的主页内容
- **且** 顶部工作区 tab 和 dock 外框不得跟随主页内容滚走

#### 场景：项目主页在 dock 面板展开时仍可滚动

- **当** 用户打开项目主页
- **且** 右侧文件或源代码管理 pane 展开
- **且** 底部终端 pane 展开
- **当** 用户在项目主页中心区域滚动
- **则** 项目主页必须仍可上下滚动
- **且** 右侧和底部 pane 不得覆盖主页内容

### 需求：顶部工作区 tab 必须只显示 icon

`消息`、`终端`、`文件`、`源代码管理` 顶部 tab 必须以 icon 工具栏形式显示，不再显示可见文字。

#### 场景：顶部 tab 不显示文字

- **当** 用户打开项目工作区
- **则** 顶部工作区 tab 必须显示 `消息`、`终端`、`文件`、`源代码管理` 对应 icon
- **且** tab 按钮内不得显示 `消息`、`终端`、`文件`、`源代码管理` 可见文字
- **且** tab 的点击区域必须保持稳定，不得因文字隐藏造成布局跳动

#### 场景：icon-only tab 保留可访问名称

- **当** 用户使用辅助技术或鼠标悬停查看顶部 tab
- **则** 每个 icon-only tab 仍必须提供对应名称
- **且** 自动化测试仍可以通过稳定的 `data-testid` 和选中态判断当前 tab 状态

### 需求：pane 控制按钮必须位于顶部

终端、文件、源代码管理等 dock pane 的移动和全屏按钮必须位于 pane 顶部控制条。

#### 场景：终端 pane 顶部显示移动和全屏按钮

- **当** 用户打开项目工作区
- **且** 底部终端 pane 正在展开
- **则** 终端 pane 的移动按钮必须显示在 pane 顶部控制条
- **且** 终端 pane 的全屏按钮必须显示在 pane 顶部控制条
- **且** 终端 pane 底部不得再显示这些控制按钮

#### 场景：右侧 pane 顶部显示移动和全屏按钮

- **当** 用户打开项目工作区
- **且** 右侧 pane 正在显示文件或源代码管理
- **则** 当前 pane 的移动按钮必须显示在 pane 顶部控制条
- **且** 当前 pane 的全屏按钮必须显示在 pane 顶部控制条
- **且** pane 底部不得再显示这些控制按钮

### 需求：pane 内部不得再提供折叠按钮

折叠和展开必须统一通过顶部对应 tab 完成，pane 内部不得再提供重复折叠入口。

#### 场景：点击终端 tab 折叠和展开终端

- **当** 底部终端 pane 正在展开
- **当** 用户点击顶部 `终端` icon tab
- **则** 底部终端 pane 必须折叠
- **当** 用户再次点击顶部 `终端` icon tab
- **则** 底部终端 pane 必须重新展开

#### 场景：点击文件或源代码管理 tab 折叠和展开右侧 pane

- **当** 右侧 pane 正在显示文件或源代码管理
- **当** 用户点击顶部当前 active 的 icon tab
- **则** 右侧 pane 必须折叠
- **当** 用户再次点击对应 icon tab
- **则** 右侧 pane 必须重新展开并恢复对应内容

#### 场景：pane 内不显示折叠按钮

- **当** 用户打开终端、文件或源代码管理 pane
- **则** pane 顶部或底部控制条中不得出现独立折叠按钮
- **且** 既有移动和全屏能力必须保持可用
