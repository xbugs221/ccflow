## 新增需求

### 需求：前端必须使用按钮和弹出框操作 wo 工作流

用户必须通过明确的按钮和弹出框选择工作流操作，不应依赖下拉选择框完成主要流程。

#### 场景：打开工作流操作弹窗

- **当** 用户进入项目主页
- **当** 用户点击 `工作流操作` 或 `启动 wo`
- **则** 页面必须打开工作流操作弹窗
- **且** 弹窗必须展示可接手的 active oz changes
- **且** active changes 必须以按钮、卡片或复选项展示
- **且** 主流程不得要求用户使用下拉选择框

#### 场景：没有可接手 change 时展示新规划入口

- **当** 项目没有可接手的 active oz change
- **当** 用户打开工作流操作弹窗
- **则** 弹窗必须展示空态
- **且** 弹窗必须提供 `发起新的规划` 操作
- **且** 页面不得显示可启动 sealed run 的误导性按钮

### 需求：前端必须支持多选 active changes 并批量启动 wo

用户可以一次选择多个 active oz changes，并为每个 change 发起独立 wo run。

#### 场景：多选两个 change 后批量启动

- **给定** 项目中存在两个可接手的 active oz changes
- **当** 用户打开工作流操作弹窗
- **当** 用户选中这两个 changes
- **当** 用户点击 `启动选中工作流`
- **则** 前端必须为每个 change 发起一次工作流创建请求
- **且** 后端必须分别执行对应的 `wo run --change <name> --json`
- **且** 页面必须展示两个启动结果
- **且** 每个成功结果必须提供进入 workflow 详情页的入口

#### 场景：批量启动时展示逐项状态

- **给定** 用户已选中多个 active oz changes
- **当** 用户点击 `启动选中工作流`
- **则** 每个 change 必须展示独立状态
- **且** 状态至少区分等待、启动中、已启动、失败
- **且** 单个 change 失败不得隐藏其他 change 的成功结果

#### 场景：单选启动成功后直接进入详情页

- **给定** 用户只选择一个 active oz change
- **当** 用户启动该 change
- **当** 后端返回 workflow
- **则** 页面必须跳转到该 workflow 详情页
- **且** 详情页必须展示 wo display lines

#### 场景：启动后刷新可接手列表

- **给定** 某个 active oz change 已成功绑定到 workflow
- **当** 用户再次打开工作流操作弹窗
- **则** 该 change 不得继续显示为可接手项

### 需求：前端必须支持发起新的规划

用户在没有现成 change 或需要先讨论需求时，可以从前端进入规划会话。

#### 场景：从工作流操作弹窗发起新规划

- **当** 用户打开工作流操作弹窗
- **当** 用户点击 `发起新的规划`
- **则** 页面必须创建或打开一个普通规划会话
- **且** 会话初始提示必须引导用户讨论问题、范围、非目标和测试策略
- **且** 本操作不得调用 `wo run`
- **且** 本操作不得创建 `.wo/runs/` 运行态目录

#### 场景：规划会话创建 oz change 后再启动 wo

- **给定** 用户已在规划会话中创建新的 oz change
- **当** 用户回到项目主页并打开工作流操作弹窗
- **则** 新 change 必须出现在可接手列表中
- **当** 用户选择该 change 并启动
- **则** 前端必须按普通 active change 流程发起 wo run

### 需求：工作流详情必须展示对应会话内容入口

启动后的 workflow 必须能从详情页进入对应阶段的会话内容。

#### 场景：点击 wo display line 进入子会话

- **给定** workflow state 中存在 `workflowDisplay.lines`
- **且** 某一行能匹配到 child session
- **当** 用户点击该行的 `start`、`review` 或 `fix` 按钮
- **则** 页面必须进入该 workflow child session 路由
- **且** 聊天区域必须展示该子会话的真实消息内容

#### 场景：未匹配 child session 的行仍可阅读

- **给定** workflow display line 未匹配到 child session
- **当** 用户打开 workflow 详情页
- **则** 该行文本必须仍然可见
- **且** 页面不得渲染无效会话跳转按钮
