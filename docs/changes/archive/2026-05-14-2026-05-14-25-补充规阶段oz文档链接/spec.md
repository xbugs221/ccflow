### 需求：规阶段必须展示 oz change 核心文档

工作流详情页必须在 `规` 阶段角色行展示当前工作流绑定的 oz change 文档链接。

#### 场景：active change 显示四个文档链接

- **当** 用户打开一个绑定 active oz change 的工作流详情页
- **则** `规` 阶段行必须展示 `proposal.md`、`design.md`、`spec.md` 和 `task.md`
- **且** 用户点击任一文档链接后必须能在现有文件查看器中打开该文档

#### 场景：规划会话缺失时仍显示文档链接

- **当** wo state 中没有可跳转的 planning session
- **且** 工作流绑定的 oz change 文档存在
- **则** `规` 阶段行仍必须展示四个文档链接
- **且** 会话缺失不得阻止用户打开文档

### 需求：归档后必须解析到最新 oz 文档路径

工作流详情页必须跟随 oz change 从 active 目录移动到 archive 目录后的最新文件位置。

#### 场景：change 归档后刷新详情页

- **当** `docs/changes/<change>/` 已移动到 `docs/changes/archive/<archived-change>/`
- **且** 用户刷新同一个工作流详情页
- **则** `规` 阶段行仍必须展示四个文档链接
- **且** 点击链接必须打开 archive 目录下的对应文档

#### 场景：archive 下存在带日期前缀目录

- **当** archive 目录名不是精确的 `<change>`，而是以 `-<change>` 结尾
- **则** 后端必须能识别该目录为当前 change 的归档目录
- **且** 前端展示的文档链接不得指向已经不存在的 active 路径

#### 场景：archive 下存在多个候选目录

- **当** archive 下存在多个可匹配同一 change 的候选目录
- **则** 后端必须选择最新修改时间的有效目录
- **且** 工作流详情页必须展示该最新目录下的四个文档链接

### 需求：其他阶段产物展示不得回退

补充 `规` 阶段文档链接不得影响已有工作流角色行行为。

#### 场景：写审修存阶段保持原有紧凑展示

- **当** 工作流同时存在 executor summary、review result、repair summary 或 delivery summary
- **则** `写`、`审`、`修`、`存` 行必须继续展示原有会话入口、计数和当前轮次产物
- **且** 不得因为 `规` 阶段多文档链接而重复展示旧轮次产物
