# Proposal: 修复会话卡片状态持久化并收敛左侧导航控件

## 目标

修复项目主页手动会话卡片右键菜单中的收藏、隐藏和恢复行为，并简化左侧项目内导航，使创建和排序只出现在项目主页控制面中。

```text
用户操作
  ├─ 项目主页会话卡片右键
  │  ├─ 收藏 / 取消收藏
  │  ├─ 隐藏
  │  └─ 显示已隐藏项 -> 取消隐藏
  │
  └─ 左侧项目内导航
     ├─ 只负责切换工作流 / 会话
     └─ 不承担排序 / 新建
```

## 背景

当前会话右键菜单的 `PUT /sessions/:id/ui-state` 请求会返回成功，但项目重新刷新后手动会话卡片没有显示收藏状态；隐藏后也无法稳定通过项目主页恢复。分析显示，状态写入使用了旧的 `sessionUiStateByPath` 顶层字段，而 `conf.json` v2 归一化保存会删除该字段，导致状态没有进入 `chat[].ui` 持久化结构。

同时，左侧导航栏的工作流和会话分组同时提供排序下拉框和新建按钮，职责与项目主页重复；项目主页中的排序下拉框存在文字和浏览器下拉箭头重叠的问题。

## 范围

- 修复普通手动会话的 `favorite`、`pending`、`hidden` UI 状态持久化和回读。
- 保证隐藏会话可以通过项目主页“显示已隐藏项”重新显示并取消隐藏。
- 移除左侧项目内导航中“需求工作流”和“手动会话”分组标题旁的排序和新建入口。
- 保留项目主页中的排序和新建入口，并修复排序下拉框文字与箭头重叠。
- 增加覆盖会话收藏、隐藏、显示已隐藏项、取消隐藏的回归测试。

## 非目标

- 不改变 workflow runner、sealed run 或 `.ccflow/runs/` 行为。
- 不重新设计项目主页卡片布局。
- 不移除项目主页的新建工作流、新建会话、多选、批量操作。
- 不改变会话 routeIndex 编号规则。

## 风险

- `conf.json` v2 中既有 `chat[].ui`，又可能残留 legacy `sessionUiStateByPath`。迁移时必须避免丢失历史 UI 状态。
- Claude、Codex 会话的 provider 识别必须正确，否则可能把状态写到错误 provider 的同名 session id。
- 左侧导航移除新建入口后，用户必须能从项目主页找到等价入口；测试需要覆盖该路径。
