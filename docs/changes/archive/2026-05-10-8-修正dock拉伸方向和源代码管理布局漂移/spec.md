## 新增需求

### 需求：右侧 dock 拉伸方向必须符合用户直觉

用户拖动右侧栏左边界时，右侧栏宽度必须按拖动方向变化。

#### 场景：向左拖动右侧栏边界

- **当** 用户打开项目工作区
- **且** 右侧 dock 正在显示 `文件` 或 `源代码管理`
- **当** 用户把右侧 dock 的 resize handle 向左拖动
- **则** 右侧 dock 宽度必须增加
- **且** 中间聊天区域必须保持可见

#### 场景：向右拖动右侧栏边界

- **当** 用户打开项目工作区
- **且** 右侧 dock 正在显示 `文件` 或 `源代码管理`
- **当** 用户把右侧 dock 的 resize handle 向右拖动
- **则** 右侧 dock 宽度必须减少
- **且** 宽度不得小于系统允许的最小值

### 需求：底部终端拉伸方向必须符合用户直觉

用户拖动终端上边界时，底部终端高度必须按拖动方向变化。

#### 场景：向上拖动终端上边界

- **当** 用户打开项目工作区
- **且** 底部 terminal dock 正在展开
- **当** 用户把底部 dock 的 resize handle 向上拖动
- **则** 底部 terminal dock 高度必须增加
- **且** 中间聊天区域必须保持可见

#### 场景：向下拖动终端上边界

- **当** 用户打开项目工作区
- **且** 底部 terminal dock 正在展开
- **当** 用户把底部 dock 的 resize handle 向下拖动
- **则** 底部 terminal dock 高度必须减少
- **且** 高度不得小于系统允许的最小值

### 需求：源代码管理不得撑开右侧 dock

`源代码管理` 必须作为右侧 dock 的内容显示，不得改变用户已经设置的右侧 dock 尺寸。

#### 场景：从文件切换到源代码管理时保持宽度

- **当** 用户打开项目工作区
- **且** 右侧 dock 正在显示 `文件`
- **且** 用户已经调整过右侧 dock 宽度
- **当** 用户点击 `源代码管理`
- **则** 右侧 dock 必须显示 Git 面板
- **且** 右侧 dock 宽度必须保持用户调整后的宽度
- **且** 中间聊天区域必须保持可见

#### 场景：反复切换右侧工具时不持续向左推进

- **当** 用户打开项目工作区
- **且** 右侧 dock 正在显示 `文件` 或 `源代码管理`
- **当** 用户多次点击 `文件` 和 `源代码管理`
- **则** 右侧 dock 不得持续向左扩大
- **且** 右侧 dock 的左边界不得因 Git 内容溢出而持续侵入聊天区域

### 需求：终端全屏能力保持不变

终端已有全屏入口，本变更不得移除或破坏该能力。

#### 场景：终端全屏入口仍然可用

- **当** 用户打开项目工作区
- **且** 底部 terminal dock 正在展开
- **则** 用户仍然可以通过现有底部 dock 控制条进入终端全屏
- **且** 退出全屏后终端必须回到底部 dock
