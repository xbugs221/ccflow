## 新增需求

### 需求：桌面工作区默认必须只选中会话 tab

进入项目或会话后，主工作区必须以会话为默认主内容，文件、终端、Git 只能作为辅助 dock 状态。

#### 场景：打开项目会话路由

- **给定** 用户本地没有有效的工作区 tab 状态
- **当** 用户打开一个项目会话路由
- **则** `会话` tab 必须处于选中状态
- **且** `文件`、`终端`、`源代码管理` tab 不得表现为主选中状态
- **且** 会话区必须显示项目会话内容或会话加载状态

#### 场景：旧 activeTab 保存为文件或终端

- **给定** 浏览器本地保存的旧 `activeTab` 是 `files`、`shell` 或 `git`
- **当** 用户重新打开项目或会话页面
- **则** 主内容必须回到 `会话`
- **且** 旧值最多只能影响对应 dock 是否展开
- **且** 页面不得默认进入空白会话区

### 需求：点击辅助 dock 按钮不得清空会话区

桌面端文件、终端、Git 按钮用于控制 dock，不应让中间会话区消失。

#### 场景：点击文件按钮后会话区仍可见

- **给定** 用户正在查看一个项目会话
- **当** 用户点击 `文件` 按钮
- **则** 右侧文件 dock 必须打开或切换到文件面板
- **且** 中间会话区必须仍然可见
- **且** 用户不得需要重新点击 `会话` 才能恢复内容

#### 场景：点击终端按钮后会话区仍可见

- **给定** 用户正在查看一个项目会话
- **当** 用户点击 `终端` 按钮
- **则** 底部终端 dock 必须打开
- **且** 中间会话区必须仍然可见
- **且** 页面不得出现空白主区域

#### 场景：点击源代码管理按钮后会话区仍可见

- **给定** 用户正在查看一个项目会话
- **当** 用户点击 `源代码管理` 按钮
- **则** 右侧 dock 必须切换到源代码管理面板
- **且** 中间会话区必须仍然可见

### 需求：普通前端页面不得自动刷新

普通项目页和会话页不得周期性重载页面或刷新项目状态导致用户界面跳动。

#### 场景：普通会话页保持稳定

- **给定** 用户正在查看普通项目会话
- **当** 用户不进行任何操作并等待数秒
- **则** 页面不得发生整页 reload
- **且** 会话区不得被清空或重置
- **且** 前端不得每秒请求项目列表来刷新普通会话页

#### 场景：运行中 workflow 仍可刷新状态

- **给定** 用户正在查看运行中的 workflow 详情
- **当** 后端 workflow 状态仍在变化
- **则** 前端可以刷新 workflow 状态
- **但** 该刷新不得影响普通会话页
- **且** 该刷新不得触发整页 reload

### 需求：底部终端栏不得显示断开和重启控制行

底部终端 dock 应只显示 dock 面板控制和终端内容，不重复显示 Shell 自己的连接控制栏。

#### 场景：打开底部终端 dock

- **给定** 用户已经打开项目工作区
- **当** 底部终端 dock 可见
- **则** 终端 dock 内不得显示 `断开连接`、`断开链接` 或 `重启` 控制行
- **且** 终端画布必须仍然可见
- **且** dock 面板自身的标题、移动和全屏控制可以保留

### 需求：终端栏必须支持新建和删除终端

用户可以在当前项目下创建多个独立终端，并关闭当前不需要的终端。

#### 场景：新建一个终端

- **给定** 用户已经打开底部终端 dock
- **当** 用户点击 `新建`
- **则** 终端栏必须新增一个终端实例
- **且** 新终端必须成为当前活动终端
- **且** 原会话区必须仍然可见

#### 场景：删除当前终端

- **给定** 用户已经打开至少一个终端实例
- **当** 用户点击 `删除`
- **则** 当前终端实例必须关闭
- **且** 前端必须关闭该终端对应的 shell 连接
- **且** 删除终端不得删除项目、会话、文件或历史记录
- **且** 原会话区必须仍然可见

#### 场景：删除最后一个终端后仍可恢复

- **给定** 用户只剩一个终端实例
- **当** 用户删除该终端
- **则** 页面必须保留可继续新建终端的入口
- **且** 工作区不得崩溃或清空会话区

### 需求：文件 tab 内部不得重复显示“文件”标题

文件 dock 外层已经提供标题时，内部文件树不应再重复显示同样文字。

#### 场景：打开文件 dock

- **给定** 用户打开项目工作区
- **当** 文件 dock 可见
- **则** dock 外层可以显示 `文件`
- **但** 文件树内部工具栏不得再次显示独立的 `文件` 标题
- **且** 搜索、刷新、新建文件、新建文件夹、上传、折叠等文件操作必须仍可使用
