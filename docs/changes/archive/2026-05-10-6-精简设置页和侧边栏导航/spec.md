## 新增需求

### 需求：设置页必须只保留核心 tab

设置页顶层导航必须只保留外观、智能体和诊断，不再展示全局 Git 配置或 API 令牌管理入口。

#### 场景：用户打开设置页

- **当** 用户打开设置弹窗
- **则** 顶层 tab 必须包含 `外观`
- **且** 顶层 tab 必须包含 `智能体`
- **且** 顶层 tab 必须包含 `诊断`
- **且** 顶层 tab 不得包含 `Git`
- **且** 顶层 tab 不得包含 `API 和令牌`

#### 场景：旧入口请求已删除 tab

- **当** 外部调用以 `git` 或 `api` 作为初始设置 tab
- **则** 设置页必须回退到 `外观`
- **且** 不得渲染已删除 tab 的内容

### 需求：外观页必须移除低价值设置组

外观页必须只保留深色模式和语言选择。

#### 场景：用户查看外观页

- **当** 用户打开 `外观` tab
- **则** 页面必须展示深色模式设置
- **且** 页面必须展示语言选择
- **且** 页面不得展示项目排序
- **且** 页面不得展示代码编辑器设置组
- **且** 页面不得展示编辑器主题、自动换行、缩略图、行号或字体大小设置

### 需求：全局 Git 设置必须被移除

ccflow 不得要求用户通过设置页写入全局 git identity。

#### 场景：用户查看设置页

- **当** 用户打开设置页
- **则** 不得出现 Git 配置入口
- **且** 不得出现用于保存全局 git name 或 git email 的表单

#### 场景：调用旧全局 Git 配置接口

- **当** 客户端请求旧的 `/api/user/git-config`
- **则** 服务端不得再提供全局 git config 读写能力
- **且** 项目工作区 Git 面板和 `/api/git` 项目级能力不得因此被移除

### 需求：API 和令牌设置必须被移除

ccflow 设置页不得继续暴露外部 API key 和 GitHub token 的持久管理面板。

#### 场景：用户查看设置页

- **当** 用户打开设置页
- **则** 不得出现 `API 和令牌` tab
- **且** 不得出现 API key 创建、删除、启停或复制入口
- **且** 不得出现设置页里的 GitHub token 持久管理入口

#### 场景：访问旧 API key 管理能力

- **当** 客户端访问旧的设置页 API key 管理路由
- **则** 服务端不得继续提供该设置页管理能力
- **且** 相关外部 API 文档不得继续作为当前设置页能力发布

### 需求：智能体页必须移除 MCP 服务器卡片

智能体页必须聚焦智能体账户和连接状态，不再提供 MCP 服务器子页。

#### 场景：用户查看智能体页

- **当** 用户打开 `智能体` tab
- **则** 页面必须展示 Codex 和 OpenCode 智能体选择
- **且** 页面必须展示账户或连接状态内容
- **且** 页面不得展示 `MCP 服务器` 子 tab
- **且** 页面不得展示新增、编辑或删除 MCP 服务器的卡片

### 需求：OpenCode 必须使用自己的 icon

OpenCode 在设置页和 provider 展示位置必须使用 OpenCode icon，不得复用 Codex 或 ChatGPT logo。

#### 场景：用户查看 OpenCode 卡片

- **当** 设置页展示 OpenCode 智能体
- **则** OpenCode 卡片必须渲染 OpenCode icon
- **且** 该 icon 不得和 Codex 卡片使用同一个 logo 组件或同一个图像资源

### 需求：OpenCode 必须展示内部已连接 provider

OpenCode 已经 connect 到内部 provider 时，前端必须展示该连接状态。

#### 场景：OpenCode 已连接 provider

- **当** 后端状态返回 OpenCode 已连接 `anthropic`
- **且** 用户打开 `智能体` tab
- **则** OpenCode 卡片必须显示已连接状态
- **且** 页面必须展示 `anthropic` 作为已连接 provider

#### 场景：OpenCode 可用但没有 provider

- **当** 后端状态返回 OpenCode CLI 可用
- **但** 没有任何内部 provider 已连接
- **则** OpenCode 卡片必须说明尚未连接 provider
- **且** 不得假装已经登录

### 需求：诊断 tab 必须中文化

中文界面下，运行诊断页面不得继续展示英文硬编码文案。

#### 场景：中文用户打开诊断页

- **当** 当前语言为 `zh-CN`
- **且** 用户打开 `诊断` tab
- **则** 页面标题必须显示为中文
- **且** 页面说明必须显示为中文
- **且** 整体状态、成功、失败、命令路径、运行目录、版本、契约能力、PATH、加载态和错误态都必须显示中文文案

### 需求：日语和韩语支持必须移除

ccflow 必须只保留中文和英文语言资源。

#### 场景：用户打开语言选择

- **当** 用户打开语言选择控件
- **则** 可选语言必须只有 English 和 简体中文
- **且** 不得展示日语
- **且** 不得展示韩语

#### 场景：旧用户保存了日语或韩语偏好

- **当** localStorage 中保存的语言为 `ja` 或 `ko`
- **则** 应用启动时必须回退到 `en`
- **且** 不得加载已删除的日语或韩语资源
- **且** 控制台不得因为缺失 ja/ko locale 资源产生初始化错误

### 需求：左侧导航栏工具按钮必须下移并移除搜索项目

左侧导航栏顶部必须只承担品牌和标题展示，项目操作工具必须移动到底部工具区，且不再提供项目搜索入口。

#### 场景：用户查看桌面端展开的左侧导航栏

- **当** 用户打开桌面端展开状态的左侧导航栏
- **则** 顶部必须展示品牌或标题
- **且** 顶部不得展示刷新项目、新建项目、搜索项目、搜索聊天记录、设置或收起侧栏按钮
- **且** 底部必须展示刷新项目按钮
- **且** 底部必须展示新建项目按钮
- **且** 底部必须展示搜索聊天记录按钮
- **且** 底部必须展示设置按钮
- **且** 底部必须展示收起侧栏按钮
- **且** 页面不得展示搜索项目按钮
- **且** 页面不得展示项目搜索输入框

#### 场景：用户查看移动端左侧导航栏

- **当** 用户打开移动端左侧导航栏
- **则** 顶部必须展示品牌或标题
- **且** 顶部不得展示项目操作按钮
- **且** 底部工具区必须展示移动端可用的刷新项目、新建项目、搜索聊天记录和设置操作
- **且** 底部工具区必须避开安全区域
- **且** 不得展示搜索项目按钮或项目搜索输入框

#### 场景：用户查看折叠的左侧导航栏

- **当** 用户把左侧导航栏收起
- **则** 折叠栏不得展示搜索项目按钮
- **且** 折叠栏仍可展示展开、搜索聊天记录和设置等当前有效操作
